# 프로젝트 현황 및 이미지 업로드 가이드 (폴더 구조 전용)

엑셀 파일 없이, **컴퓨터에 있는 폴더 구조 그대로** 웹사이트에 업로드하는 방법입니다.

## 1. 폴더 구조 준비
현재 `images` 폴더를 확인해보니 `downloaded_images` 안에 상품 폴더들이 나열되어 있습니다. 
따라서 `downloaded_images`라는 상위 이름이 **카테고리**가 되고, 그 아래 폴더들이 각각 **상품**이 됩니다.

**권장 구조:**
```text
images/
└── downloaded_images/    <-- (예: 시계, Default 등)
    ├── 10_RO151/         <-- 상품 1 (폴더)
    │   ├── front.jpg
    │   └── back.jpg
    ├── 11_RO150/         <-- 상품 2 (폴더)
    │   └── ...
    └── ...
```

## 2. 업로드 실행
다음 명령어를 실행하면, **엑셀 없이** 폴더 구조만 읽어서 자동으로 상품을 생성하고 이미지를 업로드합니다.

```bash
npx tsx scripts/migrate_folders.ts
```

*   **카테고리**: `downloaded_images`를 기본 카테고리로 인식합니다. (만약 카테고리 이름을 바꾸고 싶다면 폴더 이름을 `downloaded_images` -> `시계` 등으로 변경하세요)
*   **상품명**: 각 폴더 이름(예: `10_RO151`)이 상품명이 됩니다.
*   **가격**: 정보가 없으므로 "문의"로 저장됩니다. (나중에 관리자 페이지에서 수정)

## 3. 결과 확인
*   웹사이트를 새로고침하면 이미지가 정상적으로 표시될 것입니다.
*   Supabase Dashboard > Storage > `product-images` 버킷에서도 업로드된 파일들을 확인할 수 있습니다.
